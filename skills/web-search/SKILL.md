# 博查搜索技能

博查搜索（web-search）是一个 AI 驱动的智能搜索引擎，能够理解复杂查询、聚合多源信息，并提供带引用的准确答案。

## 🎯 核心功能

### 1. 智能搜索
- **自然语言查询**：支持完整问句搜索，无需关键词优化
- **上下文理解**：理解查询意图和背景
- **多源聚合**：整合多个网站和数据源的信息
- **实时更新**：获取最新的网络资讯

### 2. 深度研究
- **主题探索**：全面了解某个主题的各个方面
- **对比分析**：比较不同观点、产品或方案
- **趋势追踪**：了解行业发展趋势
- **事实核查**：验证信息的准确性

### 3. 专业领域
- **学术研究**：查找论文、研究报告
- **技术文档**：搜索技术资料和最佳实践
- **新闻资讯**：获取最新新闻和时事分析
- **商业情报**：市场调研和竞品分析

## 🔍 搜索技巧

### 提问方式优化

#### ✅ 推荐的提问方式

**1. 直接问完整问题**
```
❌ 不好：AI 发展
✅ 好：人工智能在 2024 年有哪些重要突破？
```

**2. 提供上下文**
```
❌ 不好：怎么做？
✅ 好：对于初创公司，如何设计可扩展的微服务架构？
```

**3. 指定时间范围**
```
❌ 不好：特斯拉新闻
✅ 好：特斯拉在 2024 年 1 月发布了哪些新产品？
```

**4. 明确需求类型**
```
❌ 不好：Python
✅ 好：推荐一些适合数据分析的 Python 库
```

#### ✅ 高级搜索技巧

**1. 对比查询**
```
"比较 React 和 Vue 在大型项目中的表现"
"对比 M1 和 M2 芯片的性能差异"
```

**2. 趋势查询**
```
"2024 年云计算行业的发展趋势"
"电动汽车市场的未来预测"
```

**3. 深度分析**
```
"分析 ChatGPT 对教育行业的影响"
"解释区块链技术的工作原理和应用场景"
```

**4. 实用指南**
```
"如何优化 MySQL 数据库性能？"
"初学者如何学习机器学习？"
```

## 📊 搜索结果理解

### 结果结构

博查搜索通常会返回：

1. **核心答案**
   - 直接回答用户问题
   - 综合多个来源的信息
   - 提供清晰的结构化内容

2. **来源引用**
   - 标注信息来源
   - 提供原始链接
   - 显示发布时间

3. **相关内容**
   - 延伸阅读建议
   - 相关话题推荐
   - 深入探索方向

### 评估结果质量

**检查清单：**
- ✅ 是否直接回答了问题？
- ✅ 信息来源是否权威可信？
- ✅ 内容是否足够新？
- ✅ 是否有多个来源佐证？
- ✅ 观点是否平衡客观？

## 🎓 最佳实践

### DO - 应该这样做

✅ **使用自然语言提问**
```
好的例子：
"解释一下量子纠缠现象"
"特斯拉 Model 3 的续航里程是多少？"
"如何在 Docker 中部署 Node.js 应用？"
```

✅ **提供足够的上下文**
```
好的例子：
"我是一个 Python 初学者，想学习数据分析，应该从哪些库开始？"
"对于日活百万的应用，如何选择数据库方案？"
```

✅ **明确信息时效性要求**
```
好的例子：
"2024 年最新的前端开发趋势是什么？"
"查找最近一个月关于 GPT-4 的新闻"
```

✅ **利用多轮对话深入**
```
第一轮：什么是 Kubernetes？
第二轮：Kubernetes 和 Docker 有什么区别？
第三轮：中小团队适合用 Kubernetes 吗？
```

### DON'T - 避免这样做

❌ **过于简短的查询**
```
不好的例子：
"AI"
"Python"
"新闻"
```

❌ **堆砌关键词**
```
不好的例子：
"机器学习 深度学习 神经网络 TensorFlow PyTorch"
应该改为：
"比较 TensorFlow 和 PyTorch 在深度学习项目中的优缺点"
```

❌ **问题过于宽泛**
```
不好的例子：
"告诉我关于编程的一切"
应该改为：
"Web 开发初学者应该学习哪种编程语言？"
```

❌ **缺乏明确目标**
```
不好的例子：
"给我一些信息"
应该改为：
"我需要了解如何优化网站加载速度"
```

## 🛡️ 信息验证

### 可信度评估

**评估维度：**

1. **来源权威性**
   - 🟢 高：官方网站、学术期刊、权威媒体
   - 🟡 中：专业博客、行业报告、知名论坛
   - 🔴 低：个人博客、社交媒体、未知来源

2. **内容时效性**
   - 🟢 最近更新：适合实时资讯和技术文档
   - 🟡 适度老化：基础理论和经典案例
   - 🔴 严重过时：需谨慎对待

3. **信息一致性**
   - ✅ 多个来源一致：可信度高
   - ⚠️ 存在分歧：需要深入判断
   - ❌ 孤证：需要额外验证

### 交叉验证方法

**步骤：**
1. 查看多个搜索结果
2. 对比不同来源的说法
3. 关注发布时间和更新频率
4. 检查作者或机构背景
5. 寻找原始数据或一手资料
